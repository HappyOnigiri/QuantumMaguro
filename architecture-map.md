# Architecture Map (Game Portal)

このドキュメントはリポジトリの構造を把握するための一次情報です。
新しい作業を始める前に、必ずこのドキュメントを読んでプロジェクトの構成を理解してください。

## 全体アーキテクチャ概要

当プロジェクトは、複数のブラウザ向けミニアプリケーション（ゲームなど）を統合・管理するための「ゲームポータル」リポジトリです。
Vite のマルチページビルドを利用し、ルートの `index.html`（ポータル）から各ミニゲームのディレクトリに遷移する構成となっています。

## ディレクトリ・ファイル構成

```text
/
├── _template/               # 新規アプリ作成用の雛形ディレクトリ（`create-app` スキルで使用）
├── public/                  # ポータル全体で共有される静的アセット
│   └── thumbnails/          # 各ゲームのポータル用サムネイル画像置き場
├── .agent/                  # 開発アシスタント（AIエージェント）用の設定ファイル群
│   ├── rules/               # プロジェクト固有のコーディングルールや運用ルール
│   └── skills/              # エージェント用自動化スクリプト（create-app, create-pr など）
├── [app-name]/              # 各ミニアプリ（ゲーム）のディレクトリ（例: quantum-maguro, mesugaki-pong）
│   ├── app.json             # アプリのメタデータ（タイトル、説明、サムネイルパスを含む）
│   ├── index.html           # アプリのエントリポイント
│   ├── tsconfig.json        # 各アプリ個別のTypeScript設定
│   └── src/                 # アプリ共通のソースコードおよび個別アセット (src/assets/)
├── index.html               # ゲームポータルのトップページ（アプリ一覧が自動生成されるエントリポイント）
├── portal.css               # トップページ専用のスタイルシート
├── package.json             # ルート全体の依存関係（各アプリ独自のpackage.jsonは持たない）
├── vite.config.ts           # 動的にアプリをスキャンし、ポータルカードやエントリポイントを構築する役割
└── Makefile                 # CI/CD、型チェック、静的解析などのユーティリティ・ツール集
```

## 主要ファイルの役割詳細

### 1. アプリケーション（ミニゲーム）ディレクトリ

アプリはルート直下の個別ディレクトリに配置されます。

- **認識の条件**: ディレクトリ内に `index.html` と `app.json` の両方が存在すること。
- ディレクトリ内の `app.json` の内容（タイトル、詳細、画像のパス等）が、ルートポータル側で読み込まれます。
- 静的アセットはアプリ内の `src/assets/` に配置し、相対パス等で参照することでViteのビルド対象となります。

### 2. ポータルの自動生成 (`vite.config.ts`)

ルートの `index.html` にある `<!-- PORTAL_CARDS -->` コメント部分に、`vite.config.ts` に構成されたプラグインが各アプリへのリンクカードを自動で生成・挿入します。
新しいゲームを追加しても、手動でポータルのHTML一覧を更新する必要はありません。

### 3. CI / 検証システム (`Makefile` & `scripts/`)

コードの検証や解析は一括して `Makefile` 経由または GitHub Actions などの運用で実施されます。

- `make ci`: TypeScirptの型チェック、HTMLのフォーマットチェック（Prettier）、Biomeでの静的解析等を実行します。
- `any`型や`@ts-ignore`の不適切な使用を防ぐ独自ルールの検査スクリプトも結合されています。

## 更新について

新しいディレクトリや主要モジュールの追加、エントリポイント・ビルドプロセスの変更など、構造変更があった場合は、必ず本ドキュメントを更新してください。
