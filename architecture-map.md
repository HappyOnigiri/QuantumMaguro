# Architecture Map

量子マグロ亭固有の理解コストが高い仕様や、誤解しやすい独自の制約のみを記載します。（Viteの標準的な挙動や一般的なファイル構成等、ソースを見れば自明なものは省略します）

## 1. 固有の制約・バリデーション
- **寿司データの整合性**: `src/data/sushi.yaml` でゲームの要となるタイピングデータ（ネタ・ローマ字）を管理します。変更時は `check_scripts/check_sushi_data.mjs` による一意性・形式チェックを通過する必要があります。
- **カスタム TypeScript ルール**: 標準のLint(Biome)とは別に、`scripts/check_ts_rules.py` によって型の甘え（`any`や`@ts-ignore`など）を厳格にチェックしています。

## 2. 独自のエイリアス解決
- **アセットパス**: `src/assets/shared/` 内のファイル（共通ロゴなど）は、HTMLやCSSにおいてもエイリアス `@shared/` を使って参照します。
- **ロジックインポート**: `src/shared/` (多言語対応の `i18n.ts` 等) へは `@shared-ts/` で参照します。

## 3. CIの実行体系
- `make ci` を実行すると、`scripts/run_ci.py` が呼ばれ、ソースの自動修正(Fix)後、上記の独自データチェックや型検査が並列実行(Check)される仕組みです。ローカルでも必ずこのプロセスを通します。
