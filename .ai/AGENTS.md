# リポジトリ基本ルール

量子マグロ亭用のリポジトリ構成に基づく運用ルールです。

## Git 操作と検証

- **自動コミット禁止**: 明示的な指示がない限り `git commit`/`push` を自動実行しないでください（`commit` スキルの使用を推奨）。
- **CIとビルドの必須化**: コード変更後は必ず `make ci` と `npm run build` を実行し、全チェックの成功を確認してください。
- **品質基準**: `any` 型や `@ts-ignore` の使用は原則禁止です。エラーや警告（特に `check_scripts/` のバリデーション）を残したままタスクを完了しないでください。

## ドキュメントと運用ルール

- **Architecture Mapの更新**: 手間がかかる調査項目や、知らずに勘違いしやすい独自仕様のみを `architecture-map.md` に記載し、常に最新に保ってください。コードや標準機能を見ればすぐに理解できる自明なことは意図的に省略し、コンパクトにします。
- **言語ルール**: 回答、思考プロセス、アーティファクト、コード内のコメントは**例外なくすべて日本語**で記述してください。
- **意図コメントの維持・付与**: リファクタリング・最適化の対象外とする意図的な実装には以下のタグコメントを使用し、変更が必要な場合は必ず理由を確認してください。
  - `[Intended] 理由`（仕様・設計上の意図）
  - `[Policy] 理由`（運用方針・制約）
  - `[Workaround] 理由`（ライブラリバグ等の回避策）
